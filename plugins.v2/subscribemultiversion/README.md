# 订阅多版本

在 MoviePilot 正常电视剧订阅下载之后，按独立规则组为相同季集追加一个目标版本。普通订阅的筛选、站点、下载器和保存路径保持不变，插件只负责额外版本的匹配与下载。

## 使用条件

- MoviePilot `>= 2.14.4, < 3`
- 已配置可用的电视剧订阅和下载器
- 已创建只接受目标版本的 MoviePilot 原生优先级规则组

## 首次配置

1. 在 **设定 -> 规则 -> 优先级规则组** 新建插件专用规则组。
2. 需要补充 Dolby Vision 时，可设置为：

   ```text
   名称：订阅多版本-DV
   媒体类型：电视剧
   媒体类别：全部
   优先级1：杜比视界（DOLBY）
   ```

3. 保存规则组编辑弹窗后，再保存“优先级规则组”区域。
4. 打开插件设置，选择该规则组，并通过“二级分类”或“电视剧订阅”限制观察范围。
5. 设置监听天数，确认范围后再启用插件。

专用规则组只在本插件中选择。不要把它加入普通订阅或 MoviePilot 的全局订阅规则组，否则普通订阅也会应用目标版本过滤条件。

## 工作方式

1. MoviePilot 普通订阅按原规则下载电视剧资源。
2. 插件接收下载事件，为对应季集创建待补任务。
3. 插件使用当前配置的专用规则组扫描 MoviePilot RSS 缓存。
4. 找到匹配资源后，沿用原订阅的站点范围、下载器、保存路径和自定义参数添加下载。
5. 暂无匹配资源时，任务保留到监听期限，并在后续成功的订阅刷新后重试。

插件不会修改普通订阅的规则组，也不会因为专用规则组设置为“电视剧 + 类别全部”而自动将其应用到其他电视剧订阅。MoviePilot 只会执行调用方明确传入的规则组名称。

## 配置项

- **启用插件**：开启事件监听和刷新观察。
- **规则组**：单选一个 MoviePilot 原生优先级规则组。
- **二级分类**：自动观察属于所选电视剧分类的订阅。
- **电视剧订阅**：额外指定需要观察的电视剧订阅。
- **监听天数**：等待目标版本发布的最长时间。

## 安全边界

- 未配置规则组、规则组不存在或规则组不适用于当前媒体时，插件不会无规则下载。
- 目标规则必须排除不需要的版本，不能只提高目标版本的优先级。
- 原下载已经满足目标规则时，不会再创建相同季集的伴生任务。
- 插件不使用媒体库缺集状态阻止伴生版本下载。
- 同一个目标任务只会成功添加一次，重启后继续保留等待状态。

## 排查

插件页面会显示任务状态、截止时间、候选标题、重试次数和最近错误。排查时先确认：

1. 专用规则组仍存在且规则内容不为空。
2. 任务对应订阅位于二级分类或电视剧订阅观察范围内。
3. RSS 缓存中存在可识别季号、集数且符合专用规则的资源。
4. 原订阅的站点和下载器仍可用。

反馈问题时请提供 MoviePilot 版本、插件版本、任务状态和已脱敏日志。请勿提交 Cookie、令牌、签名下载地址或完整 Tracker 地址。
